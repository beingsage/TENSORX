Are they all doing the same task?
No — they are not the same.

What each one actually does

AutomaticValuationModel/
A data-science project structure and pipeline.
It is not a single ready-made valuation model; it is a framework for building and training a valuation pipeline.
Best used as the project organization / ML pipeline layer.



GraphSAGE/
A graph embedding algorithm implementation.
It is not directly a price predictor.
Useful for generating graph-based features or neighborhood/transaction network signals to feed into valuation models.


House-Price-Estimator/
A tabular regression service.
It loads a pre-trained model from Real-Estate.joblib and optionally other models.
Predicts price from structured features like square footage, bedrooms, bathrooms, and location-related inputs.


Real-Estate-Valuation-Model/
A simple linear regression valuation service.
Uses UCI real estate dataset features like house age, MRT distance, and convenience store count.
It can load linear_regression_model.pkl, and if that is missing it will train from UCI_Real_Estate_Valuation.xlsx.



RealValue/
A multimodal CNN + dense model for image + structured data valuation.
Designed to use house photos plus tabular stats.
This is the most complex model and the one that most likely requires actual training on your data.



Which is best?
There is no single “best” model in this set. They are complementary.

House-Price-Estimator is best for structured tabular regression if you want a standard ensemble.
Real-Estate-Valuation-Model is a simple baseline linear model.
RealValue is the most advanced if you want to add visual/image-based valuation, but it is also the one that needs the most work and training.
GraphSAGE is best if you want graph features (neighborhood, comparable relationships, market graph structure).
AutomaticValuationModel is best if you want a reusable pipeline architecture around data prep, modeling, and evaluation.
So the best approach is likely:

use House-Price-Estimator and Real-Estate-Valuation-Model for structured-price baselines,
add RealValue for image-driven valuation,
add GraphSAGE for relational/graph signals,
and orchestrate them through a pipeline like AutomaticValuationModel.

Do they require training first or can you infer directly?
House-Price-Estimator
Can infer directly if Real-Estate.joblib is available.
If the serialized model is present, it loads at startup and can predict immediately.
If you want better performance or a new dataset, you would retrain it.
Real-Estate-Valuation-Model
It tries to load linear_regression_model.pkl.
If that file is missing, it will train from UCI_Real_Estate_Valuation.xlsx automatically.
So inference is possible if a saved model exists; otherwise it will train first.
RealValue
Designed for training.
config.yaml shows pretrained: False by default.
It expects dataset preparation, augmentation, and training via pipeline.py.
If you set pretrained: True and provide valid weights, you can use it for inference.
Otherwise, yes — you need to train it on your data first.
GraphSAGE
Not a drop-in inference model for valuation.
It needs graph data and training to produce embeddings.
Use it as a feature generator, not a final price predictor by itself.
AutomaticValuationModel
Mostly a pipeline structure; whether it can infer depends on whether you have trained model artifacts within it.
It is not inherently a ready inference service.
How much GPU VRAM is needed?
For House-Price-Estimator and Real-Estate-Valuation-Model
Very small.
These are scikit-learn models and can run on CPU easily.
GPU is not required.
For RealValue
Likely the only one needing a GPU for practical training.
With 150x150 images and a small dataset, a modest GPU is enough.
Recommended:
4–6 GB VRAM for light training and small CNNs
8–12 GB VRAM if you use larger backbone models like RegNet/EfficientNet or larger batch sizes
Inference can probably run on CPU if needed, but training is much faster on GPU.
For GraphSAGE
Depends on graph size.
Small graphs: CPU may suffice.
Larger graphs or deep GNN training: 8+ GB is safer.
It is not inherently a high-VRAM requirement unless you scale to many nodes/large feature sets.
Practical recommendation
If you want to train RealValue and also run other models at once: aim for at least 8 GB GPU.
If you only want to run tabular models and simple linear/ensemble inference: GPU is unnecessary.
Bottom line
They are not duplicates.
They are complementary:
House-Price-Estimator = structured ensemble predictor
Real-Estate-Valuation-Model = simple linear regression baseline
RealValue = image+tabular deep learning predictor
GraphSAGE = graph feature engine
AutomaticValuationModel = overall ML pipeline structure
Use pretrained artifacts if available for direct inference.
Train only when weight/model files are missing or when you want to improve the model.
For the full stack, the main GPU need is RealValue; 8 GB is a good target.
